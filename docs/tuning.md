# 参数调优指南

换音色效果受几个关键参数影响。本文给出每个参数的作用、影响与推荐值。

## 核心参数

### `--index-rate`（检索注入强度，默认 0.75）

控制 RVC 检索特征注入目标音色的比例。

| 值 | 效果 | 适用场景 |
|----|------|---------|
| 0.0 | 完全用基础模型默认音色，不像目标歌手 | 调试/对比 |
| 0.5 | 较自然，音色相似度中等 | 追求自然度 |
| 0.75 | **默认**，自然与相似度平衡 | 大多数情况 |
| 0.9~1.0 | 最像目标歌手，但易出机械感/电音 | 参考充足、追求相似度 |

**调优建议**：先用默认 0.75 听效果。太像但机械 → 调低；不够像 → 调高并增加参考歌曲数量。

### `--f0-method`（音高算法，默认 rmvpe）

| 算法 | 特点 |
|------|------|
| rmvpe | **默认**，最稳，清唱错误率最低 |
| crepe | 质量高但较慢 |
| pm | 最快，质量一般 |
| harvest | 老算法，兼容性好 |

**调优建议**：保持 rmvpe。仅在极慢设备上考虑 pm。

### `--pitch`（半音偏移，默认 0）

把整体音高上移/下移 N 个半音。**不改变旋律**，只整体平移音域。

| 场景 | 推荐值 |
|------|--------|
| 男歌 → 女声音色 | +12（高一个八度）|
| 女歌 → 男声音色 | -12 |
| 同性别人声转换 | 0 |

**注意**：偏离目标歌手自然音域过多（如 +12 以上）会导致生成器在陌生音区失真，听感像"音不准"。

## 混音参数

### `--vocal-volume` / `--accompaniment-volume`（默认 1.0 / 1.0）

人声/伴奏音量比。混合后会自动防削波归一化。

- 人声不够突出 → 提高 `--vocal-volume` 到 1.2~1.5
- 伴奏太响盖过人声 → 降低 `--accompaniment-volume` 到 0.7~0.8

## 参考歌曲选择

参考歌曲质量直接决定音色相似度：

- **数量**：3~5 首最佳；1 首覆盖面窄，易过拟合
- **内容**：尽量是干净人声段（清唱、Live 无伴奏段最好）
- **音域**：覆盖目标歌手常用音域，避免全是低音或高音
- **风格**：与目标歌曲风格接近时效果更好

## 常见问题排查

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| 机械感/电音 | index_rate 过高 | 降到 0.5~0.6 |
| 不像目标歌手 | 参考太少/不干净 | 增加参考，确保已分离干净人声 |
| 个别字音飘 | f0 提取边界错误 | 换 f0-method=crepe 重试 |
| 极端高音发虚 | 超出目标音色音域 | 调整 pitch 或换参考 |
| 人声伴奏错位 | 时长对齐异常 | 检查中间产物 converted_vocals.wav 时长 |
| 太慢 | CPU 模式 | 安装 GPU 依赖，用 `--device cuda/mps` |

## 续跑

流程中断后，加 `--resume` 自动跳过已有中间产物的阶段：

```bash
musicremix remix -i songA.wav -r ref1.wav ref2.wav -o out.wav --resume
```
